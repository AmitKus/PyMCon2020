metaclass vs inheritance

  • A metaclass is defined as “the class of a class”
  • Classes are an instance of metaclass type.

metaclass example

  • The print statement is executed during class definition creation.
class my_type(type):
    def __new__(cls, *args, **kwargs):
        print('my_type.__new__ called')
        return type.__new__(cls,  *args, **kwargs)

    def __init__(cls, *args, **kwargs):
        print('my_type.__init__ called')
        type.__init__(cls,  *args, **kwargs)
    
class B(metaclass=my_type):
    pass
my_type.__new__ called
my_type.__init__ called
a = B()

Inheritance example

  • The print statement is executed during instance creation a = B().
class my_object(object):
    def __new__(cls, *args, **kwargs):
        print('my_object.__new__ called')
        return object.__new__(cls, *args, **kwargs)

    def __init__(self, *args, **kwargs):
        print('my_object.__init__ called')
        object.__init__(self)
    
class B(my_object):
    pass

a = B()

Iterators

  • Iterator is an object which implements the iterator protocol. (iter() and next() methods).
  • Lists, tuples, dictionaries, and sets are all iterable objects.
  • They are iterable containers which you can get an iterator from.

Create an iterator

import numpy as np
class random_number:
    """
    Class generates 10 random numbers before raising StopIteration
    """
    def __iter__(self):
        self.count = 0
        return self
    
    def __next__(self):
        self.count = self.count + 1
        if self.count > 10:
            raise StopIteration
        else:
            return np.random.rand()

For loop to print random numbers

my_iter = random_number()
for i in my_iter:
    print(i)

How for loop works?

my_iter = iter(random_number())

while True:
    try:
        print(next(my_iter))
    except StopIteration:
        break

Equivalently

my_iter = random_number().__iter__()

while True:
    try:
        print(my_iter.__next__())
    except StopIteration:
        break

Advantages

  • They save resources.
  • We could get all the rand numbers without storing the entire number system in memory.
  • We can have infinite items (theoretically) in finite memory.

Generator

Generator functions allow you to declare a function that behaves like an iterator, i.e. it can be used in a for loop. Aritcle

Iterator -> generator

  • Methods like iter() and next() are implemented automatically. So we can iterate through the items using next().
  • Once the function yields, the function is paused and the control is transferred to the caller.
  • Local variables and their states are remembered between successive calls.
  • Finally, when the function terminates, StopIteration is raised automatically on further calls.

Example

def random_number(max_count=10):
    count = 0
    
    while count < max_count:
        yield np.random.rand()
        count = count + 1

Simple generator using ()

x2_gen = (x**2 for x in [1,2,3,4])
print('x2_gen is a generator object: ',x2_gen)
print('Get the values from x2_gen using next: ', next(x2_gen), next(x2_gen), next(x2_gen))
x2_gen in a generator object:  <generator object <genexpr> at 0x7f7bac9d3b88>
Get the values from x2_gen using next:  1 4 9

Apply function on generator object

x2_gen = (x**2 for x in [1,2,3,4])
print('Apply max function on the generator object: ',max(x2_gen))
Apply max function on the generator object:  16

Memoization

Memoization effectively refers to remembering (“memoization” → “memorandum” → to be remembered) results of method calls based on the method inputs and then returning the remembered result rather than computing the result again. You can think of it as a cache for method results.

Example: Without memoization

def factorial(k):
    if (k==1):
        return 1
    else:
        return k*factorial(k-1)

Example: With memoization

mem_factorial = {}
def factorial(k):
    if (k<2):
        return 1
    if k not in mem_factorial:
        mem_factorial[k] = k*factorial(k-1)
    return mem_factorial[k]

Example: Memoization class

  • Make a memoize class which takes function as argument
class Memoize:
    def __init__(self, f):
        self.f = f
        self.memo = {}

    def __call__(self, *args):
        if not args in self.memo:
            self.memo[args] = self.f(*args)
        #Warning: You may wish to do a deepcopy here if returning objects
        return self.memo[args]

Example: With function argument

def factorial(k):
    if k < 2: return 1
    return k * factorial(k - 1)

factorial = Memoize(factorial)

Example: With decorators

@Memoize
def factorial(k):
    if k < 2: return 1
    return k * factorial(k - 1)

Context manager

  • The usage of resources like file operations or database connections is very common.
  • The main problem lies in making sure to release these resources after usage.
  • They are not released then it will lead to resource leakage and may cause the system to either slow down or crash.
  • It would be very helpful if user have a mechanism for the automatic setup and teardown of resources.
  • In Python, it can be achieved by the usage of context managers which facilitate the proper handling of resources.

Example

  • Generally in other languages when working with files try-except-finally is used to ensure that the file resource is closed after usage even if there is an exception.
  • Python provides an easy way to manage resources: Context Managers. The with keyword is used.
with open('test.txt','w+') as f:
    f.write('123')
